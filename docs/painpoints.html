Executive Summary

직접선임권 트랙은 제도적 정당성(금감원 지시)에 비해 실제 활용률이 0%대에 머물러 있고, 2024년 보험업법 개정으로 손해사정서 의무 교부·인증이 도래하면서 1차 사용자(독립손해사정사)의 운영 부담이 일시에 커진 상태다. 동시에 분쟁민원이 전년 대비 31.6% 증가하고 처리기간은 2배로 늘었다. 이 갭은 곧 페인포인트의 밀도를 의미한다.

핵심 페인포인트는 (1) 진행상황 안내 부재 → 분쟁 유발, (2) 종이·대면 서류 처리, (3) 중복보험 안분과 과실 분담 설명 반복, (4) 손해사정서 교부 인증 의무로 수렴된다. B2C 청구 앱은 다수 있으나 독립손해사정사용 B2B SaaS는 거의 비어있는 화이트스페이스이며, 이것이 시장 진입 자체의 차별점이 된다.

1. 업계 구조와 직접선임권 트랙

한국의 보험금 손해사정 업무는 세 가지 트랙으로 나뉜다.

  1. 손해사정법인(보험사 위탁관계) — 보험사가 위탁한 사정사가 처리. 시장의 다수.
  2. 독립손해사정사(개인) — 라이선스 보유자가 단독 또는 소규모로 활동. 영업력 의존도 큼.
  3. 독립손해사정법인(직접선임권 트랙) — 보험계약자/피해자가 직접 선임하고 보험사가 수임료를 부담. 본 문서의 주요 1차 사용자층.

직접선임권은 "보험사가 자기들 입맛에 맞는 사정사만 위탁해 보험금을 깎는다"는 소비자 불신을 해소하기 위해 금융감독원 지시로 도입된 제도다. 2024년 2월 6일 보험업법 개정으로 공포되고 2024년 8월 7일부터 동의 기준이 시행됐으며, 시행령은 2026년 1월 1일까지 단계 발효 중이다.

제도가 사실상 사문화돼 있다

빅5 손보사 기준 2021~2024년 9월 사이 독립손해사정사 선임 비율은 매년 0%대다. 2023년 상반기 빅5 손보사 선임 요청은 57건에 그쳤고, 생·손보 합산 직접선임권 이용률은 전체의 0.2%다. 보험사 측 사정사 비동의 회신 절차, 정액형 담보를 제외하는 모범규준, 건당 보수 하락(과거 30만 → 현재 20만원 안팎) 등이 복합 작용한다.

대상 보험은 주로 배상책임보험 6종이다.

  • 일상생활배상책임보험 / 가족일상생활배상책임보험
  • 영업배상책임보험(가게·시설·행사장 등)
  • 전문인배상책임보험(사회복지사·아이돌보미 등)
  • 생산물배상책임보험
  • 의사배상책임보험(의료사고)
  • 근로자재해보험(산재 이후 별도 처리)

4. 페인포인트 상세 — 1차 사용자(독립손해사정사) 관점

P1 손해사정서 의무 교부 + 보험사 교부 인증 법령

현상
개정 보험업법 제185조에 따라 손해사정서를 작성한 보험회사는 지체 없이 보험계약자·피보험자·보험금청구권자에게 손해사정서를 교부하고 중요내용을 알릴 의무를 진다. 사업계획서에 따르면 사정사는 보험사에 교부 인증까지 수행해야 한다.
빈도
모든 종결 건. 위반 시 행정제재 가능.
시급성
2024.8.7 시행. 시행령은 2026.1.1까지 단계 발효 중.
해결 방향
손해사정서 자동 생성 템플릿 + 피해자 전자 확인 + 보험사 교부 인증 로그를 한 흐름으로 묶는다.

P3 종이·대면 서류 수기 처리 운영

현상
매 사고마다 다음 서류를 종이/대면으로 받아야 한다.
  • 보험청구서류 5종 — 손해사정사 위탁 동의서, 보험사고 처리 안내문, 보험금 지급 지연 안내문, 개인정보동의서, 손해사정서 교부 확인서
  • 손해입증자료 9종 — 진단서, 소견서, 진료기록부(초진·수술·간호·경과·닥터오더·입퇴원·통원확인서), 판독지, 진료비 영수증, 진료비 세부내역서, 약제비 영수증, 원천징수 영수증, 재직증명서
  • 종결서류 7종 — 보험금청구서, 합의서, 손해액안내확인서, 신분증, 통장사본, 가족관계증명서, 인감/본인서명사실확인서
빈도
모든 건. 사인을 일일이 받아야 함.
해결 방향
사인 자동 기입(피해자가 "동의합니다" 누르면 미리 지정된 위치에 정자체 사인 자동 기입), 손해사정사가 요청한 항목만 피해자 화면에 첨부 버튼으로 노출.

P6 현장·중간·종결 보고서 작성 부담 운영

현상
한 건당 현장보고서 1회, 중간보고서 1~3회, 종결보고서 1회. 별첨1 사례에서는 사고 수임 ~ 종결까지 약 2개월, 보고서 5회 작성.
빈도
모든 건. 직접선임권 트랙 사정사의 핵심 단점으로 사업계획서가 직접 명시.
해결 방향
총괄표·계약사항·사고경위·배상책임 검토·손해액 산정 섹션을 표준 폼으로 분해. 현장 입력 → 중간 자동 갱신 → 종결서 자동 생성.

P7 단타 소액 건 처리 비효율 사업성

현상
보험사 수수료 구조는 손해액 300만원 미만 55만원, 500만원 미만 65만원 등. 소액 단타 건이 빠른 회전 시 수익성 좋지만, 수기 보고서 작성 시간이 길어 회전이 막힘.
해결 방향
위자료 자동 산정, 안분식 자동 계산, 사고 유형별 보고서 템플릿 + 유튜브 등 외부 유입 채널과의 결합.

P11 민감정보 처리 컴플라이언스 부담 신규 법령

현상
의료기록은 개인정보보호법 제23조 민감정보로 별도 동의·안전조치 강화가 의무. 의료법상 의무기록 탐지·유출 금지가 함께 적용된다. 1인 사정사가 직접 컴플라이언스를 감당하기 어렵다.
해결 방향
표준 동의서·전송 암호화·접근 로그를 어플이 책임지면 1차 사용자에게 강한 가치.

P12 영업 의존성과 소득 양극화 사업성

현상
고용형 손해사정사 평균 연봉 약 4,500만원. 독립은 5천만~1억 이상으로 편차가 크다. "실력 아무리 뛰어나도 영업 못하면 건을 수행하지 못한다"는 사업계획서 기술과 일치.
해결 방향
지도 기반 사무실 노출, 피해자 측 유입 채널 통합, 표준 보수표 노출.

P14 정보·인맥 부족 운영

현상
업무 지식 공유, 동종업계 인맥 형성 채널이 부족. 보험사 측 사정사 혼재 가능성 때문에 폐쇄형 공간이 필요.
해결 방향
인증된 손해사정사만 가입 가능한 커뮤니티 + 주간 사례 문제 풀이.

5. 페인포인트 상세 — 2차 사용자(피해자) 관점

P2 진행상황 안내 부재 UX

현상
피해자는 케이스가 현장조사 / 법률자문 / 의료자문 / 손해액 검토 / 추가 조사 / 손해사정서 작성 / 보험사 결재 / 유사 판례 검색 / 보험계약·담보 검토 / 종결 예정 중 어느 단계인지 모른다. 보험사·사정사가 SMS·전화로만 안내.
빈도
모든 건. 분쟁민원 처리건수 +31.6%, 처리기간 30→62.5일의 2배 증가가 이 페인의 가시화.
해결 방향
사정사가 박스에 체크 → 피해자에게 단계 시각화. 종결 예정 시 예상 보험금까지 노출.

P4 중복보험 비례보상 안분 설명 반복 운영

현상
가족일상생활배상책임은 등본상 가족 1인 가입으로 처리 가능 → 한 가족이 모두 가입 시 중복보험 다수 발생. 실비도 회사 단체상해 + 개인 실비 = 2건. 손해액을 각 보험사 독립책임액 비율로 안분해야 하고, 자기부담금 차이까지 반영해야 한다.
예시
별첨1 사례: 손해액 2,133,000원을 동일 회사 3개 증권(해당 증권/모/녀) 간 독립책임액 안분, 해당 증권 지급 733,939원.
해결 방향
증권 입력 → 자기부담금 자동 반영 → 안분식과 단계 계산을 자동 출력. 피해자 화면에는 시각화된 식과 함께 노출.

P5 공동불법행위자 과실 분담 설명 부족 UX

현상
민법 제760조 공동불법행위자 구조에서, 피해자는 본인 과실(과실상계) + 공동불법행위자 책임분 차감으로 받을 금액이 줄어드는 것을 받아들이기 어렵다. 별첨1 사례: 손해액 3,048,000원 → 피해자 과실 20% 상계 → 피보험자 책임 70% × 손해액 = 2,133,000원, 학교(공동불법행위자) 책임 30% = 914,400원.
해결 방향
왜 보험에서 책임분만큼만 나가는지 자동 설명. 공동불법행위자에 대한 추가 청구(내용증명·소송 등) 안내.

P8 보험사 의료자문 신뢰 문제 신규 UX

현상
손해보험협회 26개사 기준 연간 보험금 청구 7,518만 건 중 의료자문 시행 59,026건. 그 결과 부지급 4,606건, 일부지급 13,918건. 5대 손보사 의료자문 부지급은 2023년 3,222건 → 2024년 3,779건으로 +17.3%. 부지급률 최대 10.1%.
구조적 원인
환자를 직접 보지 않고 서류로만 자문하는 구조. 자문기관(누리메디컬컨설팅 등) 대납 구조에서 환자 정보 흐름 자체에 대한 불신.
해결 방향
피해자에게 자문 동의 의사결정을 코칭. 피해자 측 손해사정사가 보험사 자문에 대해 의견 제출/대응 가능하도록 흐름 설계.

P13 피해자의 수임료 협상 시도 UX

현상
"원래 받아야 하는 돈"이라는 인식 때문에 피해자가 수임료를 깎으려는 시도가 잦다. 실손 직접선임 건당 보수는 과거 30만원에서 현재 20만원 안팎으로 하락. 비용 고려 시 수임 거부도 빈발.
해결 방향
표준 보수표(손해액 구간별 정액) + 전자 동의서 + 보험사 부담 명시로 협상 비용을 0에 수렴.

6. 페인포인트 상세 — 법·제도 환경

P9 직접선임권 인지도 부족 + 담보 범위 제한 신규 사업성

현상
빅5 손보사 기준 2021~2024.9 선임 비율 매년 0%대. 2023년 상반기 빅5 선임 요청 57건. 생·손보 합산 0.2%. 보험협회 모범규준은 적용 범위를 "실제 손해액 담보(실손·화재재물·배상책임)"로 한정해 정액형 담보 다수가 제외된다.
구조적 원인
(1) 소비자가 제도를 인지하지 못함(보험사 안내 화면 하단 링크 수준), (2) 보험사 비동의 절차가 사실상 거절 장치로 기능, (3) 정액 담보 제외로 시장 자체가 좁음.
해결 방향
피해자 측 인지 형성을 어플의 1차 임무로 설정. 사고 유형 입력 → 직접선임권 적용 가능 여부 자동 판정 → 인근 사정사 매칭.

P10 보험사·위탁 손해사정법인의 처리 지연 운영

현상
분쟁 평균 처리기간 30.1일(2019) → 62.5일(2024)로 2배. 빅5 손보사 2024년 보험금 지급 관련 피소 2,590건. 보험사·위탁 법인의 업무 처리가 늦어 직접선임 사정사의 일정도 묶인다.
해결 방향
독립사정사 측에서 직접 처리할 수 있는 비중(서류·산정·교부)을 최대화. 보험사 측 결재 대기 시간을 피해자가 가시적으로 확인.

P15 손해사정사 측 SaaS 화이트스페이스 신규 기술

현상
B2C 청구 앱은 다수 — 보맵(2015), 굿리치, 청구의신(레몬헬스케어), 라이프캐치, 실손24(보험개발원). 그러나 B2B 손해사정사 업무 SaaS는 검색상 명확한 경쟁자가 없다.
해결 방향
공급 측(독립사정사)을 정조준한 업무관리 SaaS로 진입. B2C는 부가 채널(피해자 화면)로 유지.

7. 해결 방향성과 사용자 어플의 킬러 기능 후보

법 개정 캘린더에 출시 로드맵을 정렬한다

  • 2024.8.7 — 직접선임권 동의기준 + 손해사정서 의무 교부 시행
  • 2025.10.28 — 보험업감독업무시행세칙 시행
  • 2026.1.1 — 보험업법 시행령 일부개정안 시행

이 타이밍이 1차 사용자에게 새로운 컴플라이언스 도구 수요를 만든다. 가까운 마일스톤일수록 페인포인트의 압박이 강하다.

킬러 기능 후보

  1. 손해사정서 자동 생성 + 보험사 교부 인증 트랙 (P1) — 종결 트리거 → 표준 양식(예: 동부 양식) 자동 생성 → 피해자 전자 확인 → 보험사 인증 로그.
  2. 피해자용 케이스 진행상황 실시간 대시보드 (P2) — 단계 시각화 + 종결 예정 시 예상 보험금 표시 + 분쟁 발생 지점 자동 모니터링.
  3. 전자 사인·서류 요청 흐름 (P3) — 사정사가 항목 체크 → 피해자에게 첨부 버튼 노출 → 사인 자동 기입(정자체) + 위치 사전 지정.
  4. 중복보험 안분 자동 계산기 + 시각화 (P4) — 증권/자기부담금 입력 → 독립책임액 계산 → 안분식 자동 출력. 피해자 화면에 식·차트와 함께 노출.
  5. 과실 분담 시뮬레이션 (P5) — 피보험자/피해자/공동불법행위자 과실 입력 → 차감 시퀀스 자동 시각화 → 공동불법행위자에 대한 추가 청구 가이드.
  6. 보고서 표준 폼 + 단계 자동 갱신 (P6) — 총괄표·계약사항·사고경위·배상책임 검토·손해액 산정을 모듈로 분해.
  7. 의료자문 동의 의사결정 코칭 (P8) — 사고 유형·진단명·기왕증에 따른 자문 결과 분포 예측.
  8. 지도 기반 사정사 사무실 노출 (P12) — 입점 사무실 검색 + 사고 유형 매칭.
  9. 표준 보수표·전자 위임 동의서 (P13) — 손해액 구간별 정액 + 보험사 부담 명시.
  10. 인증 사정사 전용 커뮤니티 (P14) — 주간 사례 문제 풀이 + 폐쇄형 운영.
  11. 표준 동의서·전송 암호화·접근 로그 (P11) — 민감정보 컴플라이언스 인프라.
  12. 보험내역 연계 (Phase 후순위) — 신용정보원 연동, 손찾사 류 기존 서비스 벤치마크.
